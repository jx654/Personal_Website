Machine Comprehension

To understand the purpose of this project, let’s see the definition of machine comprehension first:

A machine comprehends a passage of text if, for any question regarding that text that can be answered correctly by a majority of native speakers, that machine can provide a string which those speakers would agree both answers that question, and does not contain information irrelevant to that question.
-- Towards the Machine Comprehension of Text: An Essay

This project is aimed at teaching machine to answer multi-choice reading comprehension questions.

Dataset Description

This project uses Who-did-What, a large scale person-centered cloze dataset with multiple choice questions. This dataset has over 200,000 problems constructed from the LDC English Gigaword newswire corpus. More details can be found here.

Data Preprocessing

Since the original Who-did-What dataset is in XML format, we parsed dataset and replaced each answer in the question sentence with ’@placeholder’. What’s more, we found all person names appeared in other choices of corresponding document and did anonymization process.

from xml.etree import ElementTree as ET
from htmlentitydefs import name2codepoint
import os
import re

def clean_str(string):
  string = str(string.encode("utf8"))
  string = re.sub(r"[^A-Za-z0-9,.!?]", " ", string)
  string = re.sub(r",", " , ", string)
  string = re.sub(r"!", " ! ", string)
  string = re.sub(r"\.", " . ", string)
  string = re.sub(r"\s{2,}", " ", string)
  string = " ".join(string.split())
  return string.strip().lower()

Model

Two different models are used, one is Recurrent Neural Network(RNN) model with attentive reader, the other is Dynamic Memory Network(DMN).

Recurrent Neural Network(RNN)

    For RNN model with attentive reader, we tried LSTM layers and GRU layers combine with mlp attention function and bilinear attention function. The final results showed that GRU layers with bilinear attention function performs best on both train and validation dataset, we would choose this model.

We then added n-gram to the model. Below is the curve of accuracy changes as epoch grows:

We found that at the first few epochs, the validation accuracy of RNN models n-gram is higher than regular RNN models. However, in the end, the results of two models were almost the same.

Dynamic Memory Network(DMN)

    For Dynamic Memory Network(DMN) model, since the model is not as well-known as RNN, let’s see a brief introduction to model first:

The model contains 4 parts: input module, question module, episodic memory module and answer module. The input module reads documents while the question module reads questions and convert both to vectors. The episodic memory module updates the memory vectors based on the correlation between sentence of document and question. Finally the answer module generates the result. The training process for this model is time consuming, because the model converges very slow. In the end, we only finished 10 epochs, and the accuracy is 54.06%. Actually the result is not bad since this is a multiple-choice(num=6) question-answering problem.

Error Analysis

Due to the fact that Dynamic Memory Network(DMN) model was't well-trained enough, we used RNN model with attentive reader for error analysis. We first broke down the questions into several categories:

  • Exact Match: Sentence similar to the question can be found in the document.
  • Paraphrase: Question are expressed in a way using different words.
  • Multiple Sent: Clues for answering the question are in several different sentences.
  • Partial Clue: Only part of the clues for answering the question are directly shown in the document.
  • Hard : Questions that are even hard for native speakers.

For RNN model with attentive reader, we checked 100 document-question pairs. Among all 17 questions weren't correctly answered, there are 6 paraphrases, 3 exact matches, 5 partial clues, 1 hard or ambiguous example, and 2 multiple sentences. Actually, there may be some overlaps between the hard category and other categories.
